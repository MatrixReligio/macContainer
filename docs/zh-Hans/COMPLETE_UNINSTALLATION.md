<a id="complete-uninstallation"></a>
# 完全卸载

[英文源文档](../en/COMPLETE_UNINSTALLATION.md) · 源版本 `f94970774a25e899b7fb4a623d35c555d11f12e2`

**设置 → 运行时**提供两种有意区分的移除方式。选择前请查看说明和最新清单。

<a id="preserve-data"></a>
## 移除运行时并保留数据

此选项移除运行时组件，但保留容器镜像、存储卷、用户配置和镜像仓库凭据，适合稍后重新安装。完成信息会明确说明数据已保留；这不是完全卸载。

<a id="complete-choice"></a>
## 完全卸载选项

完全卸载会永久移除运行时以及所有产品可控的数据、凭据、缓存、测试夹具和回滚点。只有输入准确短语 `REMOVE APPLE CONTAINER` 后按钮才会启用。操作前会刷新清单、优雅停止工作负载与服务、确认进程退出、在用户上下文移除凭据、执行固定的特权清理，最后运行独立残留审计。任何缺失或无法验证的类别都会阻止成功状态。

<a id="residue-inventory"></a>
## 十五类残留

零残留合同覆盖：已审查的启动服务；运行时进程；安装收据；收据负载；Application Support 数据；用户配置；默认值域；Keychain 镜像仓库凭据；DNS 解析器条目；包过滤器锚点与规则；已下载软件包；回滚点；实体测试夹具；下载缓存；以及已审查的运行时目录。

清理使用刚刚清点的准确 URL 与清单交集，并拒绝符号链接、意外类型、路径穿越、未知所有权和过宽删除。父目录只有在已知且为空时才会移除。

<a id="verification"></a>
## 验证与恢复

清理后，独立审计器会重查全部 15 类；只有每类都证明不存在时才显示**完成**。“无法验证”按失败处理。活动中心列出剩余类别及稳定恢复键。恢复管理员权限或修复所有权后，选择**重新运行残留审计**或继续持久卸载事务。中断不会把部分卸载变成成功，也不会重复已验证的破坏性步骤。

<a id="unified-logging"></a>
## macOS 统一日志例外

已经写入 macOS Unified Logging 的历史条目由系统管理，应用无法选择性删除，因此不属于可写产品残留，也不在零残留声明内。除此之外，所有产品可控文件、服务、进程、凭据、规则、收据、软件包、缓存、测试材料和回滚点都必须移除。诊断在记录前会脱敏。失败处理参见[故障排除](TROUBLESHOOTING.md)，重新安装参见[安装](INSTALLATION.md)。
