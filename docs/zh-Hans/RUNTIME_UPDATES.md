<a id="runtime-updates"></a>
# 运行时更新

[英文源文档](../en/RUNTIME_UPDATES.md) · 源版本 `f94970774a25e899b7fb4a623d35c555d11f12e2`

MacContainer 可以发现、下载和安装 Apple container 更新，但不会因为上游出现新版本就假定其兼容。兼容性权威是内嵌的已审查目录，并由签名实体测试证明支持。

<a id="modes"></a>
## 更新模式

在**设置 → 运行时更新**选择：**仅检查**只报告兼容更新；**下载并通知**验证软件包后等待检查与管理员批准；**空闲时自动安装**还要求明确的版本化同意、既有帮助程序授权、全部兼容性门禁和完全空闲状态。活动容器、虚拟机、构建、构建器、生命周期事务或破坏性操作都会推迟安装。

<a id="fail-closed"></a>
## 内嵌允许列表与未知版本

目录将版本绑定到准确摘要、Apple 安装团队与签名方、收据、应用版本范围、适配器版本、允许的源运行时摘要、迁移和回滚类别、能力、探测集与实体证明。未知、不完整、过期、格式错误或签名错误的证据会产生不可绕过的暂停。上游监控只能创建 `UNVERIFIED` 候选 Issue，不能修改兼容性权威或自动合并。

<a id="preflight"></a>
## 安装前检查

应用会独立刷新发布元数据，验证下载大小、SHA-256、软件包签名和收据，检查应用与 macOS 版本、Apple 芯片、所需能力、允许的升级来源以及可恢复回滚点。授权前还会重新确认系统空闲；状态变忙就回到等待。

<a id="postflight"></a>
## 兼容性探测

安装后必须通过健康、容器、镜像、构建器、网络、存储卷、镜像仓库、虚拟机、磁盘占用、配置和能力共十一项域检查，且生命周期日志到达已验证终态。签名实体证明还绑定源提交、应用和运行时版本、测试计划、全部操作覆盖、零残留、基线恢复、空日志账本、nonce 与可信签名方；重放或改动都会被拒绝。

<a id="rollback"></a>
## 失败与回滚

升级前会保留目录要求的回滚内容。已批准的 1.0.0 到 1.1.0 路径保留旧软件包、配置和元数据。任一新版本探测失败都会封锁该证明，恢复旧运行时并重跑旧探测。只有旧版本重新验证后才报告成功；回滚无法验证时显示**需要恢复**，停止自动操作并保留脱敏证据，绝不在失败版本之间循环。

<a id="privacy-bandwidth"></a>
## 隐私与带宽

更新检查只请求签名目录元数据和已批准软件包，并使用条件请求与退避。凭据、容器内容、本地路径和操作历史不会上传。暂停和恢复代码参见[故障排除](TROUBLESHOOTING.md)，软件包身份参见[安装](INSTALLATION.md)。
