<a id="user-guide"></a>
# MacContainer 用户指南

[英文源文档](../en/USER_GUIDE.md) · 源版本 `f94970774a25e899b7fb4a623d35c555d11f12e2`

MacContainer 以原生界面管理已审查的 Apple container 运行时，无需终端。每项操作都会显示生效值、安全影响、进度和恢复信息。

<a id="getting-started"></a>
## 开始使用

首次启动时选择**从简单模式开始**。选择目标，填写镜像引用以及所需的工作区或端口，然后选择**检查所有值**。检查页会显示原生操作、生成值的来源及其与 Apple 默认值的差异；只有选择**运行**后才会执行。

八种内置场景不可修改，并优先采用只读根文件系统、适当的本地网络、优雅停止以及根据主机生成的 CPU 与内存建议。共享个人文件夹和嵌套虚拟化默认关闭。需要完整合同参数时可切换到高级控制，已经输入的值不会丢失。

<a id="domains"></a>
## 功能域

- **概览**显示运行时健康、兼容性、运行中的工作和最安全的后续操作。
- **容器**支持创建、启动、停止、检查、复制、导出、移除和交互会话。
- **镜像**支持拉取、推送、加载、保存、标记、检查和移除 OCI 镜像。
- **构建**配置原生构建，并在活动中心显示进度。
- **虚拟机**管理运行时虚拟机、配置、日志和默认选择。
- **网络**管理并审计网络、DNS 与包过滤器状态。
- **存储卷**管理持久存储；**镜像仓库**通过**登录**和安全输入管理凭据。空列表表示尚未保存经审核的凭据，请先登录再刷新。
- **系统**显示状态、版本、磁盘占用、日志、配置、服务和内核选择。

选择资源行可查看详情。刷新会重新加载当前域；破坏性操作会明确列出受影响资源和恢复预期。

<a id="machine-workflow"></a>
## 虚拟机工作流

打开**虚拟机**并选择**新建虚拟机**。简单模式会预先选择 **Linux 虚拟机**；检查生成的虚拟机名称、镜像、CPU 和内存，然后选择**运行**。虚拟机默认在创建后启动；创建时个人文件夹共享和嵌套虚拟化保持关闭。

默认镜像是经过启动验证并包含完整 OpenRC 初始化系统的 Alpine 3.22。MacContainer 会在首次使用时自动准备并在本地缓存。自定义虚拟机镜像属于高级选项，必须提供可用的 `/sbin/init`。选择虚拟机后使用**打开终端**；必要时应用会先启动虚拟机，再打开交互式 Linux 控制台。

容器与虚拟机是不同资源：容器是具有独立轻量 Linux 隔离的应用工作负载；虚拟机是运行自身初始化系统的持久 Linux 环境。命名卷和自定义容器网络连接到容器，虚拟机则保留自己的文件系统。

选择一台虚拟机后可使用**配置**修改 CPU 与内存，或明确启用只读个人文件夹共享和嵌套虚拟化。**启动**和**停止**作用于选中的虚拟机；**删除**前始终会显示受影响的虚拟机标识。

创建有状态容器时，可在同一向导页面选择或新建网络和命名卷。网络与存储卷页面会显示原生配置、关联容器和安全删除操作。Apple container 网络无法原地编辑，因此**创建替代网络**会复制可复用设置，并由运行时分配不重叠的子网；迁移工作负载后再删除旧网络。

<a id="registry-workflow"></a>
## 镜像仓库工作流

Apple container 不提供面向用户的通用镜像托管服务。它使用标准 OCI 镜像，因此公共镜像可匿名拉取，私有仓库可使用 Docker Hub、GitHub Container Registry、Amazon ECR、Google Artifact Registry、Azure Container Registry、Harbor 及兼容的自托管仓库。

打开**镜像仓库**并选择**登录**。输入不含镜像路径的仓库主机名、用户名，以及服务商要求的密码或访问令牌。Docker Hub 使用 `docker.io`；GitHub Container Registry 使用 `ghcr.io` 和具有相应软件包权限的个人访问令牌。MacContainer 会先验证端点，再将仅限本机的凭据存入 Apple container 拉取、推送和创建虚拟机所使用的同一安全域。匿名拉取公共镜像无需保存登录信息。

<a id="parameters"></a>
## 参数与检查

每个上游参数都有本地化标签、简要说明、完整用途、默认值、格式、重复与顺序、依赖与冲突、平台限制、安全或数据影响、示例、校验信息和恢复操作。点击字段旁的信息按钮即可查看。必填字段有标记，重复值保持输入顺序，路径与 URL 在分派前校验。最终检查会突出修改值和高影响功能；生产操作通过类型安全桥接完成，不生成 shell 命令。

<a id="templates"></a>
## 模板

打开**设置 → 默认值与模板 → 打开模板库**，选择内置模板即可查看用途和操作；内置模板不可修改。使用**新建**或**复制**创建自定义模板，在详情中编辑名称、操作和工作负载字段，然后选择**保存**。自定义模板持久保存在 Application Support 中，之后可继续选择、编辑、导出或删除。导入会迁移支持的旧架构，并在保存前阻止敏感信息或未知高影响字段；凭据与授权材料始终不会进入模板。

<a id="terminal"></a>
## 交互终端

内嵌终端会阻止远程剪贴板、链接、通知和标题更改。**分离**会保留工作负载，**终止**会发送已审查的终止信号。启用“减少动态效果”后，输出不使用装饰动画。

<a id="activity-settings"></a>
## 活动中心与设置

活动中心记录操作进度和每项结果。错误包含稳定代码、易懂原因、已脱敏诊断、重试是否安全及恢复操作。设置包含通用、运行时、运行时更新、兼容性、默认值与模板、高级和关于。切换语言会等待未保存工作及活动操作结束，再安全重启。

<a id="keyboard-accessibility"></a>
## 键盘与辅助功能

Command–N 新建场景，Command–R 刷新，Command–Shift–L 打开活动中心，Command–1 至 Command–9 切换概览到系统。主要控件均有辅助功能标签和稳定标识；键盘焦点清晰，表格与检查器无需鼠标即可使用，并支持动态字体、增强对比度和减少动态效果。

<a id="data-support"></a>
## 数据、导出与支持

默认仅本地处理，不发送分析或遥测。只有主动执行或启用的更新模式才访问网络。模板导出不是镜像或存储卷备份，请单独备份持久数据。参见[安装](INSTALLATION.md)、[运行时更新](RUNTIME_UPDATES.md)、[完全卸载](COMPLETE_UNINSTALLATION.md)和[故障排除](TROUBLESHOOTING.md)，或联系 [contact@matrixreligio.com](mailto:contact@matrixreligio.com)。
