<a id="troubleshooting"></a>
# トラブルシューティング

[英語ソース](../en/TROUBLESHOOTING.md) · ソースリビジョン `f94970774a25e899b7fb4a623d35c555d11f12e2`

エラーには安定したコード、原因、秘匿化診断、再試行の安全性、復旧操作が表示されます。アクティビティセンターで項目別記録を確認してください。

<a id="first-steps"></a>
## 最初の確認

macOS 26以降、Appleシリコン、正規Releaseのアプリであること、概要にランタイム状態が表示されることを確認し、現在の領域を一度更新します。インストール、更新、ロールバック、アンインストール中なら終端状態まで待ちます。安全と明示されない限り、特権ダイアログや破壊的操作を繰り返さないでください。

<a id="error-map"></a>
## エラーと復旧の対応

| エラーまたは復旧ID | 意味 | 安全な次の操作 |
|---|---|---|
| `error.authentication` | レジストリまたは認可情報が拒否されました。 | 認証情報を編集して一度再試行。 |
| `error.upstream-data` | 期待する上流データをデコードできません。 | メタデータを更新し、候補をインストールしない。 |
| `error.registry` | レジストリ要求が失敗しました。 | アドレスと認証情報を確認し、提示時だけ再試行。 |
| `error.helper` | 固定特権要求が拒否または利用不能です。 | 承認を確認し、アクティビティセンターを開く。 |
| `error.compatibility` | カタログ、パッケージ、証明、プローブが失敗しました。 | 互換性レポートを確認し、保留を回避しない。 |
| `error.lifecycle` | インストール、更新、復旧、削除を検証できません。 | 復旧を再開し、トランザクション記録に従う。 |
| `error.container`など | 対応するリソース操作の失敗です。 | 対象を確認し、安全表示時だけ再試行。 |
| `uninstall.recovery.*` | 残留項目が存在または検証不能です。 | アクセスを復元して残留監査を再実行。 |

パラメータ検証はフィールドの安定キーと対応する復旧キーを表示します。強調された値を修正してください。無効値は自動送信されません。

<a id="update-holds"></a>
## ランタイム更新の保留

未知のバージョン、不正カタログ、実機証明不足、未承認の更新元、パッケージID不一致、ロールバックポイント不足、プリフライト・ポストフライト失敗は自動インストールを止めます。「作業中」は全作業がアイドルになるまで待機し、「承認が必要」は自動同意またはヘルパー承認不足です。新バージョンのプローブ失敗時は証明をブロックしてロールバックします。**復旧が必要**の場合はランタイム作業を止め、明示された操作に従い、復旧データを手動削除しないでください。

<a id="uninstall-problems"></a>
## アンインストールの問題

「アンインストール未完了」は少なくとも1分類が残存または検証不能です。リゾルバやパケットフィルタには管理者権限が必要な場合があり、Keychain認証情報はユーザーコンテキストで削除します。所有者変更、シンボリックリンク、予期しない種類は意図的に拒否します。より広いディレクトリを削除せず、報告された問題を修正してください。範囲は[完全アンインストール](COMPLETE_UNINSTALLATION.md)を参照してください。

<a id="diagnostic-redaction"></a>
## 診断の秘匿化

認可ヘッダー、Bearer/Basic認証、パスワード、シークレット、APIキー、トークン、ユーザー名、URL内認証情報、非公開一時パス、ホームディレクトリ名を置換します。失敗詳細は単一行・長さ制限付きで、安全なフィールドは平文を永続化しません。共有前に詳細設定の秘匿化診断を確認し、Keychain項目、トークン、秘密鍵、コンテナデータを送らないでください。

<a id="support"></a>
## サポート

アプリ、macOS、ランタイムのバージョン、安定エラーコード、操作ID、時刻、秘匿化診断を含め、認証情報は含めないでください。[サポート](../../SUPPORT.md)または [contact@matrixreligio.com](mailto:contact@matrixreligio.com) を利用してください。脆弱性は[セキュリティポリシー](../../SECURITY.md)に従い非公開で報告します。
