<a id="user-guide"></a>
# MacContainerユーザーガイド

[英語ソース](../en/USER_GUIDE.md) · ソースリビジョン `f94970774a25e899b7fb4a623d35c555d11f12e2`

MacContainerは、レビュー済みApple containerランタイムをネイティブUIで管理します。ターミナルは不要で、各操作の適用値、安全上の影響、進行状況、復旧情報を表示します。

<a id="getting-started"></a>
## はじめに

初回起動で**シンプルモードで開始**を選びます。目的を選択し、イメージ参照と必要なワークスペースやポートを入力して、**すべての値を確認**を選択します。確認画面にはネイティブ操作、生成値の出所、Appleの既定値との差異が表示されます。**実行**を選ぶまで処理は始まりません。

8つの組み込みシナリオは変更できず、読み取り専用ルート、適切なローカルネットワーク、正常停止、ホストに応じたCPUとメモリの推奨値を優先します。ホームフォルダ共有とネストされた仮想化は明示的に有効にするまで無効です。完全な契約パラメータが必要なら詳細設定へ切り替えられ、入力済みの値は保持されます。

<a id="domains"></a>
## 機能領域

- **概要**：ランタイムの状態、互換性、実行中の作業、安全な次の操作。
- **コンテナ**：作成、開始、停止、検査、コピー、書き出し、削除、対話セッション。
- **イメージ**：OCIイメージの取得、プッシュ、読み込み、保存、タグ、検査、削除。
- **ビルド**：ネイティブビルドを設定し、アクティビティセンターで進行を確認。
- **マシン**：ランタイムVM、設定、ログ、既定マシンを管理。
- **ネットワーク**：ネットワーク、DNS、パケットフィルタ状態を管理・監査。
- **ボリューム**：永続ストレージを管理。**レジストリ**：**ログイン**と安全な入力で認証情報を管理します。空の一覧は確認済み認証情報が未保存であることを示すため、ログイン後に更新してください。
- **システム**：状態、バージョン、ディスク使用量、ログ、設定、サービス、カーネル。

リソース行を選択すると詳細を確認できます。更新は現在の領域を再読み込みし、破壊的操作は対象と復旧方針を明示します。

<a id="machine-workflow"></a>
## マシンの操作

**マシン**を開き、**新規マシン**を選択します。シンプルモードでは **Linuxマシン**が選択済みです。生成されたマシン名、イメージ、CPU、メモリを確認して**実行**を選択します。マシンは作成後に既定で起動し、ホームフォルダ共有とネストされた仮想化は作成時には無効です。

既定イメージは完全な OpenRC 初期化を含む起動検証済みの Alpine 3.22 で、初回使用時に自動準備してローカルにキャッシュします。カスタムイメージには動作する `/sbin/init` が必要です。マシンを選択して**ターミナルを開く**と、必要に応じて起動してから接続します。コンテナはアプリケーションワークロード、マシンは永続 Linux 環境です。名前付きボリュームとカスタムコンテナネットワークはコンテナに接続し、マシンは独自のファイルシステムを保持します。

マシンを1台選択して**設定**を開くと、CPUとメモリを変更し、読み取り専用ホームフォルダ共有やネストされた仮想化を明示的に有効化できます。**開始**と**停止**は選択したマシンに作用し、**削除**前には対象マシンの識別子が必ず表示されます。

<a id="registry-workflow"></a>
## レジストリの操作

Apple container は、ユーザー向けの汎用イメージホスティングサービスを提供していません。標準 OCI イメージを使用するため、公開イメージは匿名で取得でき、非公開レジストリには Docker Hub、GitHub Container Registry、Amazon ECR、Google Artifact Registry、Azure Container Registry、Harbor、および互換性のあるセルフホスト型レジストリを利用できます。

**レジストリ**を開いて**ログイン**を選択します。イメージパスを含まないレジストリのホスト名、ユーザー名、プロバイダーが要求するパスワードまたはアクセストークンを入力します。Docker Hub には `docker.io`、GitHub Container Registry には `ghcr.io` と必要なパッケージスコープを持つ個人アクセストークンを使用します。MacContainer はエンドポイントを確認してから、Apple container の取得、プッシュ、マシン作成と同じセキュリティドメインにデバイス専用の認証情報を保存します。公開イメージの匿名取得にはログイン情報の保存は不要です。

<a id="parameters"></a>
## パラメータと確認

すべての上流パラメータには、ローカライズされたラベル、概要、目的、既定値、形式、反復と順序、依存関係と競合、プラットフォーム制限、セキュリティまたはデータへの影響、例、検証メッセージ、復旧操作があります。フィールド横の情報ボタンで確認できます。必須値は明示され、反復値の順序は保持され、パスとURLは送信前に検証されます。最終確認では変更値と影響の大きい機能を示し、本番操作は型付きブリッジを使用してシェルコマンドを生成しません。

<a id="templates"></a>
## テンプレート

**設定 → 既定値とテンプレート → テンプレートライブラリを開く**で、組み込みテンプレートを選択して目的と操作を確認できます。組み込み項目は変更できません。**新規**または**複製**でカスタムテンプレートを作成し、詳細ペインで名前、操作、ワークロード項目を編集して**保存**してください。カスタムテンプレートは Application Support に永続保存され、後から選択、編集、書き出し、削除できます。読み込み時は対応する古いスキーマを移行し、保存前にシークレットや未知の高影響フィールドをブロックします。認証・認可情報は常に除外されます。

<a id="terminal"></a>
## 対話型ターミナル

組み込みターミナルはリモートのクリップボード、リンク、通知、タイトル変更をブロックします。**デタッチ**はワークロードを維持し、**終了**はレビュー済みの終了シグナルを送ります。「視差効果を減らす」有効時は装飾アニメーションを使いません。

<a id="activity-settings"></a>
## アクティビティセンターと設定

アクティビティセンターは進行状況と項目別結果を記録します。エラーには安定したコード、原因、秘匿化診断、再試行の安全性、復旧操作があります。設定には一般、ランタイム、ランタイム更新、互換性、既定値とテンプレート、詳細、情報があります。言語変更は未保存作業と実行中操作の完了を待って安全に再起動します。

<a id="keyboard-accessibility"></a>
## キーボードとアクセシビリティ

Command–Nで新規シナリオ、Command–Rで更新、Command–Shift–Lでアクティビティセンター、Command–1〜9で概要からシステムへ移動します。主要コントロールにはアクセシビリティラベルと安定IDがあり、明確なキーボードフォーカス、ポインタ不要の表とインスペクタ、Dynamic Type、コントラスト向上、視差効果軽減に対応します。

<a id="data-support"></a>
## データ、書き出し、サポート

既定ではローカル処理のみで、分析やテレメトリを送信しません。ユーザーが開始した操作または有効化した更新だけがネットワークを使用します。テンプレート書き出しはイメージやボリュームのバックアップではないため、永続データは別途バックアップしてください。[インストール](INSTALLATION.md)、[ランタイム更新](RUNTIME_UPDATES.md)、[完全アンインストール](COMPLETE_UNINSTALLATION.md)、[トラブルシューティング](TROUBLESHOOTING.md)、または [contact@matrixreligio.com](mailto:contact@matrixreligio.com) を利用してください。
