<a id="runtime-updates"></a>
# ランタイム更新

[英語ソース](../en/RUNTIME_UPDATES.md) · ソースリビジョン `f94970774a25e899b7fb4a623d35c555d11f12e2`

MacContainerはApple container更新を検出、ダウンロード、インストールできますが、新しい上流バージョンが存在するだけでは互換と判断しません。互換性の権限は、署名済み実機テスト証拠に裏付けられた埋め込みレビュー済みカタログです。

<a id="modes"></a>
## 更新モード

**設定 → ランタイム更新**で選択します。**確認のみ**は互換更新を通知して待機、**ダウンロードして通知**は承認済みパッケージを検証後に確認と管理者承認を待機、**アイドル時に自動**はさらに明示的なバージョン付き同意、ヘルパー承認、全互換性ゲート、完全なアイドル状態を要求します。コンテナ、マシン、ビルド、ビルダー、ライフサイクルトランザクション、破壊的操作が実行中なら延期します。

<a id="fail-closed"></a>
## 埋め込み許可リストと未知のバージョン

カタログはバージョンを正確なダイジェスト、Appleインストーラチームと署名者、レシート、アプリ範囲、アダプタ、許可された更新元、移行・ロールバック分類、機能、プローブ、実機証明に結び付けます。未知、不完全、古い、不正形式、署名不一致の証拠は回避できない保留になります。上流監視は`UNVERIFIED`候補Issueを作成できますが、互換性権限の編集や自動マージはできません。

<a id="preflight"></a>
## インストール前チェック

リリースメタデータを独立更新し、ダウンロードサイズ、SHA-256、パッケージ署名、レシート、アプリとmacOSのバージョン、Appleシリコン、必要機能、許可された更新元、復元可能なロールバックポイントを検証します。承認直前にもアイドルを確認し、作業が始まれば保留へ戻します。

<a id="postflight"></a>
## 互換性プローブ

インストール後は、全体の状態、コンテナ、イメージ、ビルダー、ネットワーク、ボリューム、レジストリ、マシン、ディスク使用量、設定、機能の11領域が合格し、ライフサイクルジャーナルが検証済み終端へ到達する必要があります。署名済み実機証明は、ソースコミット、アプリとランタイム、テスト計画、全操作範囲、残留ゼロ、基準復元、空の台帳、nonce、信頼署名者も結び付けます。改変や再利用は拒否されます。

<a id="rollback"></a>
## 失敗とロールバック

アップグレード前にカタログ指定の復旧データを保持します。承認済み1.0.0から1.1.0への経路では旧パッケージ、設定、メタデータを保存します。新バージョンのプローブ失敗時はその証明をブロックし、旧ランタイムを復元して旧プローブを再実行します。旧版の検証後だけ成功を表示します。ロールバックも検証できない場合は**復旧が必要**として自動操作を止め、秘匿化証拠と明確な復旧手順を保持し、失敗版間を繰り返しません。

<a id="privacy-bandwidth"></a>
## プライバシーと通信量

更新確認は署名済みカタログと承認パッケージだけを要求し、条件付き要求とバックオフを使用します。認証情報、コンテナ内容、ローカルパス、操作履歴はアップロードしません。保留・復旧コードは[トラブルシューティング](TROUBLESHOOTING.md)、パッケージIDは[インストール](INSTALLATION.md)を参照してください。
