<a id="complete-uninstallation"></a>
# 完全アンインストール

[英語ソース](../en/COMPLETE_UNINSTALLATION.md) · ソースリビジョン `f94970774a25e899b7fb4a623d35c555d11f12e2`

**設定 → ランタイム**には、意図的に異なる2つの削除方法があります。選択前に説明と最新の一覧を確認してください。

<a id="preserve-data"></a>
## データを保持してランタイムを削除

ランタイム部品を削除しつつ、コンテナイメージ、ボリューム、ユーザー設定、レジストリ認証情報を保持します。再インストール予定の場合に使用します。完了表示はデータ保持を明記し、完全アンインストールとは扱いません。

<a id="complete-choice"></a>
## 完全アンインストールの選択

ランタイムと、製品が制御する全データ、認証情報、キャッシュ、テストフィクスチャ、ロールバックポイントを完全に削除します。正確に`REMOVE APPLE CONTAINER`と入力するまでボタンは無効です。直前に一覧を更新し、ワークロードとサービスを正常停止し、プロセス終了を確認し、ユーザーコンテキストで認証情報を削除し、固定された特権クリーンアップを実行して、独立した残留監査を行います。欠落・検証不能な項目があれば成功にしません。

<a id="residue-inventory"></a>
## 15種類の残留項目

対象は、レビュー済み起動サービス、ランタイムプロセス、インストーラレシート、レシートのペイロード、Application Support、ユーザー設定、既定値ドメイン、Keychainのレジストリ認証情報、DNSリゾルバ、パケットフィルタ、ダウンロード済みパッケージ、ロールバックポイント、実機テストフィクスチャ、ダウンロードキャッシュ、レビュー済みランタイム所有ディレクトリです。

削除には直前に棚卸しした正確なURLとマニフェストの共通部分を使用し、シンボリックリンク、予期しない種類、パストラバーサル、不明な所有者、広すぎる削除を拒否します。既知の親ディレクトリも空の場合だけ削除します。

<a id="verification"></a>
## 検証と復旧

クリーンアップ後、独立監査が15種類を再確認し、すべてが不在と証明された場合だけ**完了**を表示します。「検証不能」は不在ではなく失敗です。アクティビティセンターが残存分類と安定した復旧キーを示します。管理者アクセスや所有権を修正し、**残留項目の監査を再実行**するか永続トランザクションを再開してください。中断された部分削除を成功扱いせず、検証済み破壊手順を重複実行しません。

<a id="unified-logging"></a>
## macOS Unified Loggingの除外

macOS Unified Loggingへ既に記録された履歴はシステム管理で、アプリが選択的に削除できません。そのため書き込み可能な製品残留物ではなく、残留ゼロの主張から除外されます。それ以外の製品管理ファイル、サービス、プロセス、認証情報、ルール、レシート、パッケージ、キャッシュ、テスト項目、ロールバックポイントはすべて対象です。診断は記録前に秘匿化されます。問題は[トラブルシューティング](TROUBLESHOOTING.md)、再インストールは[インストール](INSTALLATION.md)を参照してください。
