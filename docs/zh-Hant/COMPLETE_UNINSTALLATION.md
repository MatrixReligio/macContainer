<a id="complete-uninstallation"></a>
# 完整解除安裝

[英文來源文件](../en/COMPLETE_UNINSTALLATION.md) · 來源版本 `f94970774a25e899b7fb4a623d35c555d11f12e2`

**設定 → 執行階段**提供兩種刻意區分的移除方式。選擇前請查看說明和最新清單。

<a id="preserve-data"></a>
## 移除執行階段並保留資料

此選項移除執行階段元件，但保留容器映像檔、儲存卷宗、使用者設定和映像檔登錄庫憑證，適合稍後重新安裝。完成訊息會明確說明資料已保留；這不是完整解除安裝。

<a id="complete-choice"></a>
## 完整解除安裝選項

完整解除安裝會永久移除執行階段，以及所有產品可控制的資料、憑證、快取、測試資料與回復點。只有輸入正確詞句 `REMOVE APPLE CONTAINER` 後按鈕才會啟用。操作前會重新整理清單、正常停止工作負載與服務、確認程序結束、在使用者環境移除憑證、執行固定的特殊權限清理，最後執行獨立殘留項目稽核。任何缺漏或無法驗證的類別都會阻止成功狀態。

<a id="residue-inventory"></a>
## 十五類殘留項目

零殘留合約涵蓋：已審查的啟動服務；執行階段程序；安裝收據；收據負載；Application Support 資料；使用者設定；預設值網域；Keychain 映像檔登錄庫憑證；DNS 解析器項目；封包過濾器錨點與規則；已下載套件；回復點；實機測試資料；下載快取；以及已審查的執行階段目錄。

清理使用剛清點的精確 URL 與資訊清單交集，並拒絕符號連結、非預期類型、路徑穿越、未知擁有權和過度廣泛的刪除。上層目錄只有在已知且為空時才會移除。

<a id="verification"></a>
## 驗證與復原

清理後，獨立稽核器會重新檢查全部 15 類；只有每類都證明不存在時才顯示**完成**。「無法驗證」視為失敗。活動中心列出剩餘類別及穩定復原鍵。恢復管理員權限或修正擁有權後，選擇**重新執行殘留項目稽核**或繼續持久解除安裝交易。中斷不會把部分解除安裝變成成功，也不會重複已驗證的破壞性步驟。

<a id="unified-logging"></a>
## macOS 統一日誌例外

已寫入 macOS Unified Logging 的歷史項目由系統管理，應用程式無法選擇性刪除，因此不屬於可寫產品殘留，也不包含在零殘留聲明內。除此之外，所有產品可控制的檔案、服務、程序、憑證、規則、收據、套件、快取、測試資料和回復點都必須移除。診斷會在記錄前遮蔽。失敗處理請參閱[疑難排解](TROUBLESHOOTING.md)，重新安裝請參閱[安裝](INSTALLATION.md)。
