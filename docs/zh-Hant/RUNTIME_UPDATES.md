<a id="runtime-updates"></a>
# 執行階段更新

[英文來源文件](../en/RUNTIME_UPDATES.md) · 來源版本 `f94970774a25e899b7fb4a623d35c555d11f12e2`

MacContainer 可以發現、下載和安裝 Apple container 更新，但不會因為上游出現新版本就假定相容。相容性權威是內嵌且已審查的目錄，並由簽章實機測試證明支援。

<a id="modes"></a>
## 更新模式

在**設定 → 執行階段更新**選擇：**僅檢查**只回報相容更新；**下載並通知**驗證套件後等候檢查與管理員核准；**閒置時自動安裝**還需要明確的版本化同意、既有輔助程式授權、全部相容性關卡和完全閒置狀態。進行中的容器、虛擬機器、建置、建置器、生命週期交易或破壞性操作都會延後安裝。

<a id="fail-closed"></a>
## 內嵌允許清單與未知版本

目錄將版本綁定到精確摘要、Apple 安裝團隊與簽章者、收據、應用程式版本範圍、介接器版本、允許的來源執行階段摘要、移轉與回復類別、能力、探測集和實機證明。未知、不完整、過期、格式錯誤或簽章錯誤的證據會造成無法略過的暫停。上游監控只能建立 `UNVERIFIED` 候選 Issue，不能修改相容性權威或自動合併。

<a id="preflight"></a>
## 安裝前檢查

應用程式會獨立重新整理發佈中繼資料，驗證下載大小、SHA-256、套件簽章與收據，檢查應用程式及 macOS 版本、Apple 晶片、必要能力、允許的升級來源和可復原回復點。授權前還會再次確認系統閒置；狀態變忙便回到等候。

<a id="postflight"></a>
## 相容性探測

安裝後必須通過健康狀態、容器、映像檔、建置器、網路、儲存卷宗、映像檔登錄庫、虛擬機器、磁碟用量、設定和能力共十一個領域的檢查，且生命週期日誌到達已驗證終態。簽章實機證明還綁定來源提交、應用程式與執行階段版本、測試計畫、全部操作覆蓋、零殘留、基準復原、空日誌帳本、nonce 和可信簽章者；重播或變更都會被拒絕。

<a id="rollback"></a>
## 失敗與回復

升級前會保留目錄要求的回復內容。已核准的 1.0.0 到 1.1.0 路徑保留舊套件、設定和中繼資料。任何新版本探測失敗都會封鎖該證明，還原舊執行階段並重新執行舊探測。只有舊版本再次驗證後才回報成功；回復無法驗證時顯示**需要復原**，停止自動操作並保留已遮蔽證據，絕不在失敗版本間循環。

<a id="privacy-bandwidth"></a>
## 隱私權與頻寬

更新檢查只要求簽章目錄中繼資料和已核准套件，並使用條件式要求與退避。憑證、容器內容、本機路徑和操作記錄不會上傳。暫停及復原程式碼請參閱[疑難排解](TROUBLESHOOTING.md)，套件身分請參閱[安裝](INSTALLATION.md)。
