# MacContainer 사용 설명서

[영문 소스](../en/USER_GUIDE.md) · 소스 리비전 `f94970774a25e899b7fb4a623d35c555d11f12e2`

MacContainer는 검토된 Apple container 런타임을 네이티브 UI로 관리합니다. 터미널이 필요 없으며 각 작업의 적용 값, 안전 영향, 진행 상황, 복구 정보를 표시합니다.

<a id="getting-started"></a>
## 시작하기

처음 실행할 때 **단순 모드로 시작**을 선택합니다. 목표를 선택하고 이미지 참조 및 필요한 작업 공간 또는 포트 정보를 입력한 다음 **모든 값 검토**를 선택합니다. 검토 화면은 네이티브 작업, 생성된 값의 출처, Apple 기본값과의 차이를 보여 줍니다. **실행**을 선택하기 전에는 아무 작업도 시작되지 않습니다.

8개의 기본 제공 시나리오는 변경할 수 없으며 읽기 전용 루트 파일 시스템, 적절한 로컬 네트워크, 정상 종료, 호스트 기반 CPU 및 메모리 권장값을 우선합니다. 홈 폴더 공유와 중첩 가상화는 명시적으로 켜기 전까지 꺼져 있습니다. 전체 계약 매개변수가 필요하면 고급 제어로 전환할 수 있고 입력한 값은 유지됩니다.

<a id="domains"></a>
## 기능 영역

- **개요**: 런타임 상태, 호환성, 실행 중인 작업, 가장 안전한 다음 조치.
- **컨테이너**: 만들기, 시작, 중지, 검사, 복사, 내보내기, 제거, 대화형 세션.
- **이미지**: OCI 이미지 가져오기, 푸시, 로드, 저장, 태그, 검사, 제거.
- **빌드**: 네이티브 빌드 구성과 활동 센터 진행 상황.
- **머신**: 런타임 VM, 구성, 로그, 기본 머신 관리.
- **네트워크**: 네트워크, DNS, 패킷 필터 상태 관리 및 감사.
- **볼륨**: 영구 저장소 관리. **레지스트리**: **로그인**과 보안 입력으로 자격 증명을 관리합니다. 빈 목록은 검토된 자격 증명이 저장되지 않았다는 뜻이므로 로그인한 다음 새로 고치십시오.
- **시스템**: 상태, 버전, 디스크 사용량, 로그, 구성, 서비스, 커널 선택.

리소스 행을 선택해 세부 정보를 확인합니다. 새로 고침은 현재 영역을 다시 불러오며 파괴적 작업은 대상 리소스와 복구 방식을 명확히 표시합니다.

<a id="machine-workflow"></a>
## 머신 작업 흐름

**머신**을 열고 **새 머신**을 선택합니다. 단순 모드에서 **Linux 머신**이 미리 선택됩니다. 생성된 머신 이름, 이미지, CPU, 메모리를 검토한 다음 **실행**을 선택합니다. 머신은 생성 후 기본적으로 시작되며 홈 폴더 공유와 중첩 가상화는 생성 중 비활성화됩니다.

머신 하나를 선택하고 **구성**을 열어 CPU와 메모리를 변경하거나 읽기 전용 홈 폴더 공유와 중첩 가상화를 명시적으로 활성화할 수 있습니다. **시작**과 **중지**는 선택한 머신에 적용되며 **삭제** 전에는 영향을 받는 머신 식별자가 항상 표시됩니다.

<a id="registry-workflow"></a>
## 레지스트리 작업 흐름

Apple container는 사용자를 위한 범용 이미지 호스팅 서비스를 운영하지 않습니다. 표준 OCI 이미지를 사용하므로 공개 이미지는 익명으로 가져올 수 있고, 비공개 레지스트리는 Docker Hub, GitHub Container Registry, Amazon ECR, Google Artifact Registry, Azure Container Registry, Harbor 및 호환되는 자체 호스팅 레지스트리와 함께 사용할 수 있습니다.

**레지스트리**를 열고 **로그인**을 선택합니다. 이미지 경로가 없는 레지스트리 호스트 이름, 사용자 이름, 공급자가 요구하는 암호 또는 액세스 토큰을 입력합니다. Docker Hub에는 `docker.io`를 사용하고 GitHub Container Registry에는 `ghcr.io`와 필요한 패키지 범위가 있는 개인 액세스 토큰을 사용합니다. MacContainer는 엔드포인트를 확인한 후 Apple container가 이미지 가져오기, 푸시 및 머신 생성에 사용하는 동일한 보안 도메인에 기기 전용 자격 증명을 저장합니다. 공개 이미지의 익명 가져오기에는 저장된 로그인이 필요하지 않습니다.

<a id="parameters"></a>
## 매개변수 및 검토

모든 업스트림 매개변수에는 현지화된 레이블, 간단한 설명, 목적, 기본값, 형식, 반복과 순서, 종속성과 충돌, 플랫폼 제한, 보안 또는 데이터 영향, 예, 검증 메시지, 복구 작업이 있습니다. 필드 옆 정보 버튼에서 확인할 수 있습니다. 필수 필드는 표시되고 반복 값 순서는 유지되며 경로와 URL은 전달 전에 검증됩니다. 최종 검토는 변경된 값과 영향이 큰 기능을 표시합니다. 프로덕션 작업은 형식화된 브리지를 사용하며 셸 명령을 생성하지 않습니다.

<a id="templates"></a>
## 템플릿

**설정 → 기본값 및 템플릿 → 템플릿 보관함 열기**에서 기본 제공 템플릿을 선택해 목적과 작업을 확인할 수 있습니다. 기본 제공 항목은 수정할 수 없습니다. **새로 만들기** 또는 **복제**로 사용자 지정 템플릿을 만들고 세부 정보 패널에서 이름, 작업, 워크로드 필드를 편집한 다음 **저장**하십시오. 사용자 지정 템플릿은 Application Support에 영구 저장되며 나중에 선택, 편집, 내보내기 또는 삭제할 수 있습니다. 가져오기는 지원되는 이전 스키마를 마이그레이션하고 저장 전에 비밀 정보나 알 수 없는 고영향 필드를 차단합니다. 자격 증명과 권한 부여 자료는 항상 제외됩니다.

<a id="terminal"></a>
## 대화형 터미널

내장 터미널은 원격 클립보드, 링크, 알림, 제목 변경을 차단합니다. **분리**는 워크로드를 유지하고 **종료**는 검토된 종료 신호를 보냅니다. 동작 줄이기가 켜져 있으면 장식 애니메이션을 사용하지 않습니다.

<a id="activity-settings"></a>
## 활동 센터 및 설정

활동 센터는 진행 상황과 항목별 결과를 기록합니다. 오류는 안정된 코드, 원인, 삭제 처리된 진단, 재시도 안전 여부, 복구 작업을 제공합니다. 설정에는 일반, 런타임, 런타임 업데이트, 호환성, 기본값 및 템플릿, 고급, 정보가 있습니다. 언어 변경은 저장되지 않은 작업과 실행 중인 작업이 끝난 후 안전하게 다시 실행합니다.

<a id="keyboard-accessibility"></a>
## 키보드 및 손쉬운 사용

Command–N은 새 시나리오, Command–R은 새로 고침, Command–Shift–L은 활동 센터, Command–1~9는 개요부터 시스템까지 이동합니다. 주요 제어에는 손쉬운 사용 레이블과 안정 ID가 있고 명확한 키보드 포커스, 포인터 없이 사용하는 표와 검사기, Dynamic Type, 대비 증가, 동작 줄이기를 지원합니다.
