<a id="complete-uninstallation"></a>
# 완전 제거

[영문 소스](../en/COMPLETE_UNINSTALLATION.md) · 소스 리비전 `f94970774a25e899b7fb4a623d35c555d11f12e2`

**설정 → 런타임**에는 의도적으로 다른 두 가지 제거 방법이 있습니다. 선택 전에 설명과 최신 목록을 확인하십시오.

<a id="preserve-data"></a>
## 데이터를 유지하고 런타임 제거

런타임 구성 요소를 제거하면서 컨테이너 이미지, 볼륨, 사용자 구성, 레지스트리 자격 증명을 유지합니다. 나중에 다시 설치할 때 사용합니다. 완료 메시지는 데이터 유지 사실을 명시하며 완전 제거로 취급하지 않습니다.

<a id="complete-choice"></a>
## 완전 제거 선택

런타임과 제품이 제어하는 모든 데이터, 자격 증명, 캐시, 테스트 픽스처, 롤백 지점을 영구적으로 제거합니다. 정확한 문구 `REMOVE APPLE CONTAINER`를 입력할 때까지 버튼이 비활성화됩니다. 직전에 목록을 새로 확인하고 워크로드와 서비스를 정상 종료하며 프로세스 종료를 확인하고 사용자 컨텍스트에서 자격 증명을 제거하고 고정된 권한 정리를 수행한 뒤 독립적인 잔여 항목 감사를 실행합니다. 누락되거나 검증할 수 없는 범주가 있으면 성공으로 표시하지 않습니다.

<a id="residue-inventory"></a>
## 잔여 항목 15개 범주

검토된 시작 서비스, 런타임 프로세스, 설치 영수증, 영수증 페이로드, Application Support 데이터, 사용자 구성, 기본값 도메인, Keychain 레지스트리 자격 증명, DNS 리졸버, 패킷 필터, 다운로드한 패키지, 롤백 지점, 실제 기기 테스트 픽스처, 다운로드 캐시, 검토된 런타임 소유 디렉터리를 포함합니다.

정리는 방금 조사한 정확한 URL과 매니페스트 교집합을 사용하고 심볼릭 링크, 예상치 못한 형식, 경로 탐색, 알 수 없는 소유권, 지나치게 넓은 삭제를 거부합니다. 알려진 상위 디렉터리도 비어 있을 때만 제거합니다.

<a id="verification"></a>
## 검증 및 복구

정리 후 독립 감사기가 15개 범주를 다시 확인하고 모두 없음이 증명될 때만 **완료**를 표시합니다. “검증할 수 없음”은 없음이 아니라 실패입니다. 활동 센터는 남은 범주와 안정된 복구 키를 표시합니다. 관리자 접근 또는 소유권을 수정한 뒤 **잔여 항목 감사 다시 실행**을 선택하거나 영구 제거 트랜잭션을 재개하십시오. 중단된 부분 제거를 성공으로 바꾸지 않고 검증된 파괴 단계를 반복하지 않습니다.

<a id="unified-logging"></a>
## macOS 통합 로깅 제외

macOS Unified Logging에 이미 기록된 과거 항목은 시스템이 관리하며 앱이 선택적으로 삭제할 수 없습니다. 따라서 쓰기 가능한 제품 잔여물이 아니며 잔여 항목 없음 주장에서도 제외됩니다. 그 외 제품 관리 파일, 서비스, 프로세스, 자격 증명, 규칙, 영수증, 패키지, 캐시, 테스트 항목, 롤백 지점은 모두 제거 대상입니다. 진단은 기록 전에 삭제 처리됩니다. 문제는 [문제 해결](TROUBLESHOOTING.md), 재설치는 [설치](INSTALLATION.md)를 참조하십시오.
