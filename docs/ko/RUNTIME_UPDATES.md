<a id="runtime-updates"></a>
# 런타임 업데이트

[영문 소스](../en/RUNTIME_UPDATES.md) · 소스 리비전 `f94970774a25e899b7fb4a623d35c555d11f12e2`

MacContainer는 Apple container 업데이트를 검색, 다운로드, 설치할 수 있지만 새 업스트림 버전이 존재한다는 이유만으로 호환된다고 판단하지 않습니다. 호환성 권한은 서명된 실제 기기 테스트 증거로 뒷받침되는 포함된 검토 카탈로그입니다.

<a id="modes"></a>
## 업데이트 모드

**설정 → 런타임 업데이트**에서 선택합니다. **확인만**은 호환 업데이트를 알리고 대기합니다. **다운로드 후 알림**은 승인된 패키지를 검증한 다음 검토와 관리자 승인을 기다립니다. **유휴 시 자동 설치**는 명시적인 버전별 동의, 기존 도우미 승인, 모든 호환성 게이트, 완전 유휴 상태를 추가로 요구합니다. 컨테이너, 머신, 빌드, 빌더, 수명 주기 트랜잭션 또는 파괴적 작업이 활성 상태이면 설치를 미룹니다.

<a id="fail-closed"></a>
## 포함된 허용 목록과 알 수 없는 버전

카탈로그는 버전을 정확한 다이제스트, Apple 설치 팀과 서명자, 영수증, 앱 범위, 어댑터, 허용된 업데이트 원본, 마이그레이션 및 롤백 분류, 기능, 검사 집합, 실제 기기 증명에 연결합니다. 알 수 없거나 불완전하거나 오래되었거나 잘못된 형식 또는 서명의 증거는 우회할 수 없는 보류가 됩니다. 업스트림 모니터는 `UNVERIFIED` 후보 Issue만 만들 수 있으며 호환성 권한을 편집하거나 자동 병합할 수 없습니다.

<a id="preflight"></a>
## 설치 전 검사

릴리스 메타데이터를 독립적으로 새로 고치고 다운로드 크기, SHA-256, 패키지 서명, 영수증, 앱 및 macOS 버전, Apple Silicon, 필수 기능, 허용된 업데이트 원본, 복원 가능한 롤백 지점을 검증합니다. 승인 직전에도 유휴 상태를 확인하며 작업이 시작되면 대기로 돌아갑니다.

<a id="postflight"></a>
## 호환성 검사

설치 후 전체 상태, 컨테이너, 이미지, 빌더, 네트워크, 볼륨, 레지스트리, 머신, 디스크 사용량, 구성, 기능의 11개 영역이 통과하고 수명 주기 저널이 검증된 종료 상태에 도달해야 합니다. 서명된 실제 기기 증명은 소스 커밋, 앱 및 런타임, 테스트 계획, 전체 작업 범위, 잔여 항목 없음, 기준 상태 복원, 빈 원장, nonce, 신뢰 서명자도 연결합니다. 변경 또는 재사용은 거부됩니다.

<a id="rollback"></a>
## 실패 및 롤백

업그레이드 전에 카탈로그가 요구하는 복구 데이터를 유지합니다. 승인된 1.0.0에서 1.1.0 경로는 이전 패키지, 구성, 메타데이터를 보존합니다. 새 버전의 검사 실패 시 해당 증명을 차단하고 이전 런타임을 복원한 뒤 이전 검사를 다시 실행합니다. 이전 버전이 검증된 후에만 성공을 표시합니다. 롤백도 검증할 수 없으면 **복구 필요**로 표시하고 자동 작업을 중지하며 삭제 처리된 증거와 명확한 복구 절차를 보존합니다. 실패한 버전 사이를 반복하지 않습니다.

<a id="privacy-bandwidth"></a>
## 개인정보 보호 및 대역폭

업데이트 확인은 서명된 카탈로그와 승인된 패키지만 요청하며 조건부 요청과 백오프를 사용합니다. 자격 증명, 컨테이너 내용, 로컬 경로, 작업 기록을 업로드하지 않습니다. 보류 및 복구 코드는 [문제 해결](TROUBLESHOOTING.md), 패키지 ID는 [설치](INSTALLATION.md)를 참조하십시오.
